About the Habitat-Scanner Tool

Welcome to the **Habitat-Scanner Tool**.

This tool empowers airport operators, wildlife managers, and safety experts to rapidly identify and map potential wildlife attractants around any aerodrome or location worldwide.

By fusing live data from **OpenStreetMap (OSM)** and **Google Earth Engine (GEE)** satellite imagery, this tool scans the selected radius for critical habitats. It generates an interactive map, a KML file, and a CSV report detailing features like water bodies, landfills, wastewater plants, and agricultural areas.

This provides a rapid, data-driven baseline for your Wildlife Hazard Management Programme (WHMP) and helps prioritize on-ground surveys.

Methodology

This tool implements a two-pronged data-sourcing strategy:

  • OpenStreetMap (OSM) Data: The tool queries the Overpass API for known, human-mapped features. This includes landuse=landfill, amenity=slaughterhouse, natural=water, and other tags associated with wildlife attractants.
  • Google Earth Engine (GEE) Data: To find features not mapped by OSM, the tool analyzes ESA WorldCover satellite data. It automatically identifies and vectorizes Water Bodies (Class 80) and Cropland (Class 40).

The results from both sources are combined, filtered by your "Minimum Hazard Area" setting, and presented as a single, comprehensive dataset.
